# docs 索引重建规则

老项目接入 setup-workflow 时，如果 `docs/` 中已经存在文档，需要检查索引是否完整。

## 需要检查

每个 `docs/` 下的目录都应该有自己的 `index.md`。

每个 `index.md` 只描述当前目录层级：

- 当前目录的直接子目录。
- 当前目录的直接文档。

不要级联展开父目录或子孙目录内容。

## 需要重建的情况

- 缺少 `index.md`。
- `index.md` 没有列出本层子目录。
- `index.md` 没有列出本层文档。
- 文档条目没有一句话描述。

## 描述要求

文档的一句话描述必须由 Agent 阅读文档内容后编写。

可以使用脚本列出目录结构，但不能用脚本或文件名猜测描述。
